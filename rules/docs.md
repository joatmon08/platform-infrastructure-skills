## Critical Requirement

**ALWAYS update documentation when making changes to the repository.**

### When Making Any Changes

Whenever you:
- Add new scripts, files, or directories
- Modify existing functionality or architecture
- Change development conventions or standards
- Add new projects or components
- Update dependencies or technologies
- Modify pipeline flows or processes
- Change security practices or requirements
- Fix bugs or implement features

### You MUST Update

1. **AGENTS.md** - Update relevant sections to reflect changes
2. **Project README.md** - Update project-specific documentation if applicable
3. **Root README.md** - Update if adding/removing projects or changing repository structure
4. **Any other related documentation** - Update any docs that reference changed components

### Before Completing Tasks

- Verify that AGENTS.md accurately reflects all changes made
- Ensure all documentation stays synchronized with code
- Update examples and usage instructions if behavior changed
- Document new conventions or patterns introduced

This ensures future agents and team members have accurate, up-to-date guidance.

### External Links

- Use the Developer documentation link for HashiCorp products: `https://developer.hashicorp.com`. Do not
  use the `.io` websites (e.g., `terraform.io`).
